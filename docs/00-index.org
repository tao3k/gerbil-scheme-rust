* Documentation contract

Design documents state ownership, invariants, non-goals, and checked
boundaries before implementation detail.  Roadmaps separate planned work,
acceptance gates, current evidence, and remaining gaps.  A completed checkbox
is not evidence by itself; commands, test receipts, or release artifacts must
support completion claims.
