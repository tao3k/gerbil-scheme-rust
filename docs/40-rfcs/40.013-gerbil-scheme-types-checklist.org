#+TITLE: RFC 40.013 Gerbil Scheme Types Binding Checklist
#+DATE: 2026-07-20
#+STATUS: Draft

* Purpose

This checklist tracks the remaining work required for =gerbil-scheme-rust= to
become a general-purpose Gerbil Scheme Rust bindings library, rather than a
Marlin-specific extraction.

The checklist is intentionally type-first. Marlin integration should consume
this crate only after the native Scheme value surface, ownership rules,
scenario coverage, and performance receipts are clear enough to be reused.

* Current evidence baseline

Current Rust ABI evidence:

- =gerbil-scheme-sys= exposes =GerbilStatus=, =GerbilBorrowedUtf8=,
  =GerbilRuntimeOpaque=, =GerbilValueHandle=, and =GerbilI64Callback=.
- =gerbil-scheme= exposes safe =GerbilRuntime=, =GerbilUtf8=,
  =GerbilValue=, =GerbilI64Callback=, =GerbilI64CallbackAbi=, and
  =NativeError=.
- Scalar native operations currently cover i64 identity/addition, i64 parity,
  and i64 comparison.
- =native-value-surface= scenario covers safe UTF-8, opaque value handles, and
  callback trampoline behavior.
- =source-surface-sync= and =backed-value-family-surface= scenarios cover the
  Scheme ASP source-surface contract for result/error taxonomy and every
  currently sys/safe-backed value-family shape.

Current Scheme source-surface evidence:

- =scheme/asp/native-surface.ss= is still the ASP-projected workaround for the
  source-surface view.
- The current projected selector set is exactly 18 selectors:
  =gerbil_scheme_rust_abi_version=, =gerbil_scheme_rust_runtime_init=,
  =gerbil_scheme_rust_runtime_cleanup=, =gerbil_scheme_rust_identity_i64=,
  =gerbil_scheme_rust_add_i64=, =gerbil_scheme_rust_is_even_i64=,
  =gerbil_scheme_rust_compare_i64=,
  =gerbil_scheme_rust_runtime_handle_shape=,
  =gerbil_scheme_rust_status_shape=, =gerbil_scheme_rust_i64_shape=,
  =gerbil_scheme_rust_bool_shape=,
  =gerbil_scheme_rust_comparison_shape=,
  =gerbil_scheme_rust_utf8_shape=,
  =gerbil_scheme_rust_value_handle_shape=,
  =gerbil_scheme_rust_i64_callback_shape=,
  =gerbil_scheme_rust_native_value_shape=,
  =gerbil_scheme_rust_native_error_shape=, and
  =gerbil_scheme_rust_native_result_shape=.
- Direct provider projection of the real =scheme/native.ss= FFI/export owner is
  not yet closed and remains a provider/source-surface TODO.
- No current Scheme selector exists for nil, void, f64, char, symbol, keyword,
  pair, list, vector, procedure, owned/rooted values, bytevectors, ports,
  continuations, or advanced Scheme objects.

* Type surface checklist

** Runtime and lifecycle

- [X] Process-global runtime safe owner: =GerbilRuntime=.
- [X] Thread-affinity check for runtime methods.
- [X] ABI version check before exposing runtime handle.
- [X] Drop-time native cleanup path.
- [ ] Explicit runtime re-entry policy document: what is legal after cleanup,
  what is process-final, and what must fail closed.
- [ ] Scenario for wrong-thread access and runtime-finalized behavior.
- [ ] Public receipt type for runtime initialization, ABI id, ABI version, and
  native module path.

** Status, error, and result

- [X] Stable raw status enum: =GerbilStatus=.
- [X] Safe native error enum: =NativeError=.
- [X] Unknown status preservation in status contract tests.
- [X] Panic containment for i64 callback trampoline.
- [ ] Public =GerbilResult<T>= alias or typed result wrapper.
- [?] Scheme-side result shape projection for success/error payloads is present
  as ASP source-surface taxonomy, but no public =GerbilResult<T>= wrapper exists
  yet.
- [ ] Error mapping table from Gerbil exception classes to stable Rust errors.
- [ ] Scenario that validates status/error/result projection together.

** Nil, void, eof, and unspecified

- [ ] Raw ABI representation for EOF object if exported.
- [ ] Raw ABI representation for unspecified values.
- [ ] Safe Rust marker types for nil/void/eof/unspecified.
- [ ] Tests proving these sentinels are distinguishable and do not alias null.

* Ownership and safety checklist

- [ ] Every public wrapper states whether it is borrowed, owned, rooted, or
  process-global.
- [ ] Every raw pointer has a nullability rule.
- [ ] Every borrowed pointer has a retention rule.
- [ ] Every callback has a panic containment rule.
- [ ] Every runtime-bound value has thread-affinity semantics.
- [ ] Every value crossing FFI has an error/status mapping.
- [ ] No public safe API exposes =unsafe extern "C"= directly.
- [ ] Unsafe blocks include narrow safety comments at the actual boundary.
- [ ] Scenario coverage exists for null, invalid value, panic, wrong thread, and
  lifecycle-finalized failures.
