#+title: Gerbil Scheme Rust Type Matrix
#+date: 2026-07-20

Every matrix row must satisfy the bridge layering, opaque handle, provenance,
and merge-policy rules in
[[file:40-rfcs/40.014-gerbil-scheme-rust-bridge-engineering-standard.org][RFC 40.014]].

* Purpose

This matrix tracks the public Scheme value surface exposed by
=gerbil-scheme-rust=.  Each row must eventually connect four receipts:

- C ABI representation
- Rust public type
- ownership and lifetime contract
- unit, scenario, and performance coverage

* Current ABI value surface

| Scheme value kind | C ABI representation | Rust public type | Ownership contract | Current coverage |
|-------------------+----------------------+------------------+--------------------+------------------|
| status | =repr(i32)= enum | =GerbilStatus= | by value | unit + native result tests |

* Safe handle-backed value views

The safe =gerbil-scheme= crate now exposes handle-backed views for value
families that require runtime ownership, allocation, interning, traversal, or GC
rooting before they can become owned Rust values:

| Scheme value kind | Safe Rust view | Backing representation | Explicit non-goal |
|-------------------+----------------+------------------------+-------------------|
| symbol | =SchemeSymbol= | non-zero =GerbilValueHandle= | no allocation or interning |
| keyword | =SchemeKeyword= | non-zero =GerbilValueHandle= | no allocation or interning |
| pair | =SchemePair= plus =GerbilValue::is_pair=/=pair_car=/=pair_cdr=/=pair_parts= | non-zero =GerbilValueHandle= | =is_pair= and =pair_car=/=pair_cdr=/=pair_parts= are runtime-backed only for =SchemeObjectExport=; =UntrustedRaw= and =RuntimeSentinel= remain fail-closed |
| list | =SchemeList= plus =GerbilValue::is_list=/=is_null= | non-zero =GerbilValueHandle= | =is_list= and =is_null= are runtime-backed for Scheme object exports |

These types preserve handle identity and reject zero handles.  Pair/list
status wrappers call the sys ABI and report explicit =NativeResult= status
errors for unbacked handles, so the public API can be consumed before
runtime-backed traversal is wired in.  They do not dereference unmanaged memory
and do not claim ownership or GC rooting.

* Gaps

** Runtime-backed pair/list predicate gate

Pair/list/null predicates currently keep the raw sys ABI fail-closed for
unbacked handles.  A direct Gambit C predicate experiment can classify a fake
non-zero Rust address as =Ok(false)= instead of =InvalidValue=; that weakens the
safe wrapper contract because =GerbilValue::from_raw= can still construct a
handle from arbitrary non-zero words.

Runtime-backed predicate support therefore needs a provenance boundary before
the public safe API changes behavior:

- current =GerbilValue::from_raw= handles are explicitly tagged as
  =GerbilValueProvenance::UntrustedRaw=;
- =UntrustedRaw= proves only non-zero shape, not live Gambit/Gerbil object
  ownership, GC reachability, type, or traversal safety;
- runtime-sentinel handles are explicitly tagged separately from =UntrustedRaw=
  before any pair/list behavior changes;
- Scheme object exports are explicitly tagged as
  =GerbilValueProvenance::SchemeObjectExport= and remain borrowed/unrooted;
- =is_null= routes to Gerbil =null?= only for =SchemeObjectExport= handles;
- =is_pair= and =is_list= route to Gerbil =pair?= and =list?= only for
  =SchemeObjectExport= handles;
- add a runtime-owned value constructor that can only be produced by an
  initialized =GerbilRuntime= or a Gerbil export;
- keep arbitrary =from_raw= handles fail-closed unless they carry runtime
  provenance;
- route =pair?=, =list?=, and =null?= through Gerbil/Gambit only for
  provenance-backed handles;
- route =car=, =cdr=, and =pair_parts= through Gerbil only for
  =SchemeObjectExport= handles while keeping =UntrustedRaw= and
  =RuntimeSentinel= fail-closed;
- do not add recursive list traversal in this gate;
- keep Marlin as a downstream consumer of this upstream contract, not an owner
  of Scheme runtime traversal semantics.

** Runtime conversion gaps

- Bignum is not yet represented separately from fixnum.
- Symbol, keyword, pair/list construction, vector construction, bytevector
  allocation, and procedure invocation do not yet have live runtime conversion
  APIs.
- GC rooting and handle lifetime rules are not yet encoded as Rust types.

** Scenario and performance gaps

- The current matrix has unit coverage for ABI shape, but not enough downstream
  scenario receipts for each value family.
- Required scenario receipts should include:
  - scalar by-value call overhead
  - string and bytevector borrow/validation overhead
  - vector/list traversal overhead
  - callback dispatch overhead
  - runtime init/cleanup lifecycle overhead

* Next acceptance target

The next closed increment should add a =gerbil-scheme-sys= scenario suite that
uses the Rust harness scenario benchmark contract and reports sub-millisecond
timing keys for the value-family cases above.
