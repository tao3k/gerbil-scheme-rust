* Implemented Surface

- =gerbil-scheme-sys=: versioned C ABI types and the native scalar export.
- =gerbil-scheme=: safe process-backed fallback for version, evaluation, and
  scalar projection.  It is not described as the native backend.
- =GerbilRuntime=: safe, exclusive, thread-affine ownership of an initialized
  in-process Gerbil/Gambit runtime, with ABI validation and cleanup on drop.
- =gerbil-scheme-build=: opaque source emission, artifact projection, and
  checked =gxc= invocation.
- =scheme/native.ss=: a scalar C export implemented with Gerbil's official
  =:std/foreign= / Gambit =c-define= path.
- =scheme/gerbil_scheme_rust.h=: versioned C header matching the Rust sys
  declarations; the runtime ABI reports version 1.
- =build.ss=: canonical Gerbil build truth.
- =native/runtime.c=: minimal setup/cleanup shim around Gambit's supported
  =___setup= / =___cleanup= lifecycle and the generated module linker.
- =gerbil-scheme-sys/build.rs=: Cargo-side packaging of canonical =build.ss=
  output into a native archive; it does not reimplement Gerbil compilation.
- Bazel: outer orchestration of locked Cargo gates and the canonical Gerbil
  native gate.

* Safe In-process Boundary

The safe =GerbilRuntime= constructor serializes the process-global lifecycle,
calls the native setup shim, validates ABI version 1, and only then returns a
handle.  The handle is neither =Send= nor =Sync=, checks its owner thread, owns
the loaded module lifetime, and runs cleanup when dropped.  Rust integration
tests call =gerbil_scheme_rust_add_i64(40, 2)= in process and receive 42; they
also reject a second live owner.  Cleanup finalizes Gambit for the process, so
later initialization is rejected with a stable status instead of attempting an
unsupported restart.

The process-backed implementation remains a separately named fallback during
the expansion of the native API.  It is not used as evidence for the in-process
native acceptance gate.

* Next Native Expansion Gate

1. Add status-returning native entry points before introducing operations whose
   Scheme bodies can raise exceptions.
2. Contain Scheme exceptions and Rust panics before either crosses the C ABI.
3. Introduce owned value handles only together with explicit release functions
   and generation checks tied to =GerbilRuntime=.
4. Keep every new safe method on the same exclusive, thread-affine handle until
   the official runtime contract supports a broader threading model.
